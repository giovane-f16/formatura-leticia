# SDD Index

Este diretorio organiza o desenvolvimento orientado por especificacao do projeto iniciado em `prompt.md`.

## Ordem de leitura

1. `prompt.md` - entrada original do produto.
2. `01-product-brief.md` - visao, publico e proposta.
3. `02-requirements.md` - requisitos funcionais e nao funcionais.
4. `03-user-flows.md` - fluxos principais de navegacao e interacao.
5. `04-frontend-design-spec.md` - direcao visual, layout e componentes.
6. `05-technical-spec.md` - arquitetura React e dados editaveis.
7. `06-implementation-plan.md` - fases de implementacao.
8. `07-task-breakdown.md` - backlog inicial.
9. `08-acceptance-criteria.md` - criterios de aceite.
10. `09-content-and-assets.md` - textos, fotos e placeholders.
11. `10-decision-log.md` - decisoes registradas.

## Regra SDD

Toda mudanca relevante deve atualizar primeiro a especificacao correspondente. A implementacao deve poder ser validada contra os criterios de aceite.
